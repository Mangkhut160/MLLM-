电池审核项目架构总览

主路线：Prompt 基线 -> SFT/LoRA -> GRPO -> 阈值校准

离线训练链路
原始数据

1. data.csv + brand_new + charge_new

2. 清洗缺图/空标签/路径不一致样本

3. 组装统一样本：文本字段 + 两张图 + 结构化审核输出

4. Prompt 验证与冷启动解释样本构造

5. Qwen2.5-VL + LoRA/SFT

6. GRPO 优化：结果 + 标签 + 短理由

7. 验证集评测与阈值校准

在线审核链路
用户提交审核

输入：品牌 / 日期 / 电压 / 容量 + 两张图片

模型输出：各审核点结果 + 最终结果 + 标签 + 简短理由

决策层：读取关键审核点通过概率与总阈值

自动通过：关键点全部通过且置信度达标

人工复核：任一关键点失败、低置信度、格式异常

结果沉淀：复核结果回流，继续更新训练数据

审核输出结构
{
  "brand_check": "通过/驳回",
  "date_check": "通过/驳回",
  "voltage_capacity_check": "通过/驳回",
  "spec_check": "通过/驳回",
  "image_check": "通过/驳回",
  "final_result": "通过/人工复核",
  "reject_tags": ["..."],
  "reason": "30字内短理由"
}

  • 先完成可演示的 SFT 审核 MVP
  • 重点验证结构化审核和阈值自动放行

阶段二

  • 用 GRPO 强化短理由、标签和结果稳定性
